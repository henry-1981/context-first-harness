# Medical Device RA/QA Thinking Frame

사용자의 의료기기 RA/QA 사고 프레임을 자동 로드하는 게이트웨이 스킬입니다. 프레임 본문은 wiki에 source of truth로 보관하고, 본 스킬은 트리거·요약·에스컬레이션 규율만 담습니다.

## 가장 먼저 할 일

활성화 직후 다음 파일을 Read 합니다.

- `wiki/pages/medical-device-ra-qa-thinking-frame.md`

이 파일이 1부 명시지 4축과 2부 HITL 3영역의 전체 원문입니다. 실제 판단은 wiki 원문을 기준으로 수행하며, 본 스킬은 인덱스와 규율만 제공합니다. 원문과 본 스킬의 기재가 충돌하면 **wiki 원문이 우선**입니다.

## 1부 — 명시지 4축 (자동 적용)

사용자 확인 경유 없이 작동합니다. 각 축의 세부 적용 규정은 wiki 원문 해당 섹션을 참조합니다.

1. **스코프 게이트** — 의료기기 인허가·전략 맥락 판정 + 계약·업무수행내역서 범위 확인. 프로젝트 폴더에 자료가 있다는 사실만으로 scope 확대 금지.
2. **정보 취급 규율** — 1차 원문 끝까지 읽기(법률 조항은 단서·예외·면제까지), 레퍼런스 제품 사양을 클라이언트 제품 확정 사실로 전제 금지, 미확인 항목은 `[확인 필요]` 또는 `[사용자 리뷰 대상]` placeholder.
3. **가설 기반 전략 + Risk Register** — 전략은 작업 가설로 서술, 검증 경로 병기, 상태 변화는 단순 삭제가 아니라 Withdrawn/Resolved/Reopened 전환으로 기록.
4. **사실 기반 밀도 + 산출물 Voice** — 밀도(사실로 빼곡히 채운 논증)와 voice(~습니다 포멀, 긴 연결 문장, counter-position 인정 후 1) 2) 3) inline 번호, technical term 영어 단독)는 분리 불가. **산출물 전용**이며 대화 레지스터로 역류 금지.

## 2부 — HITL 3영역 (방향 확인 필수)

다음 판단이 발생하면 **자동 진행 금지**. 반드시 사용자에게 방향 확인을 요청합니다.

- **영역 A — 영향성 평가의 범위·체계성**: 사실 하나가 확인되거나 방법론이 바뀌었을 때 다른 전략·전술에 ripple이 미칠지 재평가 필요 여부. "작은 변화에도 필수적"의 수준이 암묵지이므로 LLM이 독자 판정 금지.
- **영역 B — 층위 식별 언어의 일반성**: 전략 vs 전술, 공통 기반 vs 차별화, 이분법 대신 같은 축 위 레벨 차이, 기존 허가 범위 내 부속 기능 vs 신규 claim 경계 판단. 특정 프로젝트 bias 위험이 커서 다른 제품·사안에 기계적 이식 금지.
- **영역 C — 협업 구조의 방향/위임 경계**: "방향"에 속해 사용자가 직접 쥐어야 할 판단인지 "세부"로 LLM 위임 가능한지. 경계선이 암묵지.

## HITL 에스컬레이션 루틴

HITL 영역을 건드리는 순간 다음 순서로 응답합니다. 사용자가 매번 처음부터 설명하지 않도록 잠정 판단안을 먼저 제시한 뒤 방향 확인을 요청하는 것이 핵심입니다.

1. 현재 상황이 A/B/C 중 어느 영역인지 명시
2. LLM이 잠정적으로 떠올린 판단안을 짧게 제시
3. 사용자에게 방향 확인 요청
4. 필요 시 `clarify:unknown` 또는 `clarify:vague` 사용 제안

방향 확인 결과 암묵지가 명시지로 추출되면 wiki 원문의 2부에서 1부로 승격하고, `wiki/log.md`에 `update | medical-device-ra-qa-thinking-frame` 엔트리를 기록합니다.

## Red Flags — 합리화 감지 시 STOP

다음 생각이 떠오르면 즉시 멈추고 해당 축/영역의 규율을 따릅니다.

| 합리화 | 실제 |
|--------|------|
| "이 정도 변화는 작으니 영향성 평가 생략해도 됨" | 체계성 수준이 암묵지다. LLM이 범위를 독자 판정하지 않는다. 영역 A 에스컬레이트. |
| "이전 프로젝트에서 본 패턴이니 이 프로젝트에도 그대로 적용" | 프로젝트 bias 위험. 층위 식별은 검증되지 않은 귀납이다. 영역 B 에스컬레이트. |
| "세부 문구 수정이니 사용자 확인 없이 진행" | "방향 vs 세부" 커팅 라인이 암묵지다. 경계선이면 영역 C 에스컬레이트. |
| "유사제품 스펙이 있으니 클라이언트 제품도 이럴 것" | 축 2 위반. 레퍼런스는 "유사제품의 경우 ~" 형태로만 인용하고 클라이언트 제품 사양은 사용자 확인본만 사용. |
| "법률 조항 제1항만 확인해도 판정 가능" | 축 2 위반. 제2항 이하의 단서·예외·면제까지 확인한 후에만 판정. |
| "산출물 voice가 대화에서도 자연스러울 것" | 축 4 위반. ~습니다 포멀은 산출물 전용. 대화는 별도 레지스터(~요/~어/~지) 유지. |
| "전략이 확정이니 Risk Register 업데이트 불필요" | 축 3 위반. 전략은 작업 가설이다. 상태 전환을 기록해야 맥락 복원이 가능하다. |
| "프로젝트 폴더에 자료가 있으니 스코프 안일 것" | 축 1 위반. 계약·업무수행내역서 기준으로 다시 판정. 자료 존재 ≠ 스코프. |
| "이 정도 추측은 사용자도 이해할 것" | 축 2 위반. 미확인은 `[확인 필요]`로 명시. 빈칸을 추측으로 메우지 않는다. |

## 활성화 조건

다음 중 하나라도 감지되면 본 스킬을 로드합니다.

- 디렉토리 진입: 의료기기 인허가 관련 `compliance/` 하위
- 키워드: SaMD, 허가, 식약처, 임상시험, 수가, NHT, IFU, Decision Support, 기술문서, 인허가 전략, 품목분류, 급여
- 사용자가 의료기기 RA/QA 사안 개시를 직접 선언

활성화 후 가장 먼저 **축 1 스코프 게이트**를 통과하는지 판정합니다. 의심 시 사용자에게 확인을 요청합니다.
